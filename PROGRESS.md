## Что это за проект
«Без Б» (Блог Белоглазова) — публичный инвестпортфель в Telegram.
Цель: превратить бот-трекер в продукт с платной подпиской + AI-агенты для
контента. Личный бренд «инвестиции без буллшита». Канал @BezBlogfin,
бот @BezzBot_bot (id 7707472432, ADMIN_ID 503720103).

Активы: крипта, акции США, золото. Покупки раз в 2 недели (DCA).
Оплата подписки решено брать в USDT. Хостинг — дешёвый VPS. AI-агент пишет
посты ЧЕРНОВИКОМ на ревью владельцу (не автопубликация).

Стратегия: НЕ монетизировать сразу — сначала публичная история + аудитория
(сейчас подписчиков и трек-рекорда нет). Продаём систему/прозрачность, а не
«доходность 50%».

## Сделано в этой сессии (2026-06-11)

  `.venv`, `__pycache__`).
- Старый токен отозван у @BotFather, новый вписан в `.env` (проверен getMe).
- В `run_polling` добавлен `drop_pending_updates=True`.
- Бот перезапущен и работает (фоновый процесс на ПК — умрёт при выключении).

### 2. Решение: переход с кнопок на Telegram Mini App
Кнопочный UI не нравится. Делаем веб-приложение внутри Telegram.
Архитектура: бот = пусковая кнопка + пуши; бизнес-логика (`portfolio.py`/
`quotes.py`/`benchmark.py`) переиспользуется через API; фронт React+Vite+TS;
хранилище позже JSON→SQLite; хостинг VPS+nginx+HTTPS.

### 3. Прототип Mini App — готов (папка `miniapp/`)
React + Vite + TypeScript. Тёмная тема (#0e1117 / #26a69a / #ef5350).
Шапка «Bez ₿» (логотип Bitcoin вместо буквы B). Все эмодзи → SVG-иконки
(`src/components/Icons.tsx`).

**5 экранов** (нижняя навигация), все собраны:
- **Портфель** — баланс в $/₽, двойная доходность, индекс, donut, позиции,
  «Повторить портфель» (+ пустое состояние).
- **Динамика** — SVG-графики (стоимость vs вложено, индекс), «Без Б против
  рынка» столбиками, переключатель периодов.
- **Журнал** — лента-дневник сделок с причинами.
- **Сделки** (только владелец) — сетка Купить/Продать/Пополнить/Вывести + чипы
  причин.
- **Профиль** — карточка, блок Премиум (990 ₽/мес), «Поделиться»,
  «Подписаться на канал».

### 4. Этап 0 — реальные данные подключены

## Как запустить локально (напоминание)
Из папки `bezb_bot`:
```powershell
# 1) Бот (опционально)
.\.venv\Scripts\python.exe bot.py

# 2) API (бэкенд Mini App)
.\.venv\Scripts\python.exe -m uvicorn api:app --port 8000

# 3) Фронт Mini App
cd miniapp ; npm run dev    # откроется на http://localhost:5173/
```

## Деплой Mini App на GitHub Pages — чек-лист
0. Установить git: `winget install --id Git.Git -e` (или GitHub Desktop).
1. Создать **публичный** репо на GitHub (имя любое).
2. В папке проекта: `git init` → `git branch -M main` → `git add .` →
   `git commit -m "..."` → `git remote add origin <URL>` → `git push -u origin main`.
3. Settings → Pages → Source = **GitHub Actions**.
4. Деплой стартует сам после push (или Actions → Run workflow). Ждать зелёного.
5. URL: `https://ЛОГИН.github.io/РЕПО/` — откроется на мок-данных.
6. @BotFather → `/newapp` (бот @BezzBot_bot) → Web App URL = адрес из п.5;
   `/setmenubutton` → тот же URL → подпись кнопки.
7. (Позже) задеплоить API на VPS/Render → задать переменную репо
   `VITE_API_BASE` → перезапустить workflow → реальные данные.

> Pages = только статика; `api.py` там не работает → пока мок, пока нет своего API.
