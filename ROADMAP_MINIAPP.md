# «Без Б» — дорожная карта Telegram Mini App

Документ описывает переход от кнопочного бота к **Telegram Mini App** (веб-приложение
внутри Telegram) для проекта «Без Б — инвестиции без буллшита».

---

## Зачем Mini App, а не кнопки

| Кнопочный бот (сейчас) | Mini App (цель) |
|---|---|
| Меню из кнопок, всё «лесенкой» | Полноценный экран: карточки, табы, анимация |
| Графики — статичные PNG | Интерактивные графики (зум, наведение, периоды) |
| Формы = цепочка сообщений | Нормальные формы с валидацией на лету |
| Выглядит «как все боты» | Свой бренд, тёмная тема, премиум-ощущение |
| Тяжело показать подписку/премиум | Гейтинг экранов, оплата, апселл — естественно |

Бот **не исчезает**: его роль сужается до (1) кнопки запуска Mini App и
(2) пуш-уведомлений (сделки, недельный отчёт). Бизнес-логика остаётся прежней.

---

## Целевая архитектура

```
Telegram ──► Бот (PTB)              ──┐  кнопка web_app + пуши
            (запуск Mini App,         │
             уведомления)             │
                                      ├─► общая бизнес-логика
Telegram ──► Mini App (фронт, HTTPS) ─┤   portfolio / quotes / benchmark
            React + графики            │
                │ initData (HMAC)      │
                ▼                      │
            API (FastAPI/uvicorn)  ────┘
                │
                ▼
            Хранилище (SQLite)
```

- **Фронт** — SPA на React + Vite + TypeScript, хостится статикой по HTTPS.
  Графики — `lightweight-charts` (TradingView, бесплатно — «профессиональный»
  вид) + donut на свой вкус. SDK Telegram WebApp для темы/кнопок/хаптики.
- **API** — FastAPI поверх СУЩЕСТВУЮЩИХ `portfolio.py` / `quotes.py` /
  `benchmark.py` (они чистый Python, без привязки к Telegram — переиспользуем).
  Авторизация: проверка `initData` (HMAC по токену бота) → узнаём user id →
  отличаем владельца (ADMIN_ID) от зрителя.
- **Хранилище** — миграция JSON → **SQLite**. Причина: бот и API — два процесса,
  а сейчас данные в одном JSON-файле; два писателя устроят гонку. SQLite это
  снимает и готовит почву под мультиюзера/подписку.
- **Хостинг** — VPS + nginx + Let's Encrypt (TLS обязателен: Telegram открывает
  Mini App только по HTTPS). nginx раздаёт статику фронта и проксирует `/api`.

## Дизайн интерфейса (5 экранов, нижняя навигация)

Бренд: тёмный фон `#0e1117`, акцент бирюзовый `#26a69a`, минус — `#ef5350`.

1. **Портфель** (главная)
   - Большая карточка баланса: `$ / ₽`, прибыль `% в ₽` и `% в $` (фирменная
     двухвалютная фишка), значение «Индекс Без Б».
   - Donut состава (интерактивный).
   - Список позиций карточками: тикер, стоимость, ср.→тек. цена, P/L %.
   - CTA «Повторить портфель».
2. **Динамика**
   - Интерактивный график стоимости vs вложено; линия индекса; переключатель
     периодов.
   - «Без Б против рынка» — столбики (S&P / BTC / золото / вклад / доллар).
3. **Журнал** — лента сделок с причинами = публичный дневник решений
   (витрина доверия): карточка каждой сделки (купил/продал, актив, сумма,
   причина, доля в портфеле).
4. **Сделки** (только владелец) — формы купить/продать/пополнить/вывести
   с подтверждением и выбором причины; триггер поста в канал.
5. **Профиль/Подписка** — статус подписки, премиум-функции, карточка-результат
   для шеринга, кнопка «Подписаться на канал», (позже) прогресс геймификации.

Используем нативные возможности WebApp: адаптация под тему Telegram, `MainButton`
для главного действия, `BackButton`, хаптика, шеринг, разворот на весь экран.

## Дорожная карта (этапы, каждый даёт рабочий результат)

### Этап 0 — Фундамент (инфраструктура)
- VPS + домен + HTTPS (нужно и для бота 24/7, и для Mini App).
- Вынести бизнес-логику в API (FastAPI), переиспользуя текущие модули.
- Миграция хранилища JSON → SQLite (убрать гонку бот↔API).
- Авторизация через Telegram `initData`.
- **Результат:** API отдаёт сводку портфеля; бот и API живут на VPS.

### Этап 1 — Каркас Mini App (только просмотр)
- Скелет фронта (React+Vite), брендинг, тёмная тема, нижняя навигация.
- Экран «Портфель» (баланс, donut, позиции) — read-only.
- Кнопка в боте `/start`, открывающая Mini App (`web_app`).
- Деплой: статика + API за nginx.
- **Результат:** любой подписчик открывает красивый портфель в Telegram.

### Этап 2 — Графики и журнал
- Интерактивные графики (динамика, индекс, сравнение с рынком).
- Лента «Журнал решений» — витрина истории.
- «Повторить портфель» как интерактивный калькулятор капитала.
- **Результат:** есть весь контент-смысл проекта в удобном виде.

### Этап 3 — Управление (владелец)
- Формы купить/продать/пополнить/вывести с подтверждением.
- Журнал причин при сделке; пост в канал из Mini App.
- **Результат:** автор ведёт портфель целиком из Mini App, бот-кнопки можно
  убрать.

### Этап 4 — Подписка и премиум
- Гейтинг премиум-экранов; оплата USDT (CryptoCloud/Cryptomus).
- Премиум: мгновенные пуш-уведомления о сделках, «машина времени»
  (портфель на любую дату), AI-разбор «твой портфель vs Без Б».
- **Результат:** первая выручка.

### Этап 5 — Геймификация и рост
- Недельные стрики дисциплины, ачивки, прогресс к финансовой цели.
- Шеринг карточек, реферальная программа, связка с Tradernet (IB).
- **Результат:** удержание и органический рост.

---

## Принцип последовательности
Инфраструктура → удобный просмотр → богатый просмотр → управление →
монетизация → удержание. Каждый этап самостоятелен и оставляет продукт в
рабочем состоянии; нельзя монетизировать (Этап 4) раньше, чем есть что
показывать (Этапы 1–2) и публичная история.
